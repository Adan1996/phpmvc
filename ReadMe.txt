Apa itu MVC ?
Pola arsitektur pada perancanganperangkat lunak berorientasi objek, tujuan MVC sendiri ialah untuk
memisahkan antara tampilan, data dan proses

Kenapa MVC ?
- Organisasi dan Struktur kode
- Pemisahan logic dan tampilan
- Perawatan kode
- Implementasi konsep oop yang sudah dipelajari
- Digunakan oleh banyak Web Aplication Framework

Framework MVC :
-----------------------------------------------------
|Bahasa         |Framework                          |
-----------------------------------------------------
|PHP            |Codeigniter, Laravel, Yii, dll     |
|Java           |Spring MVC, JSF, Struts, dll       |
|Python         |Django, CherryPy, dll              |
|Ruby           |Ruby On Rails, Sinatra, dll        |
|Javascript     |AngularJS, React, Backbone.js, dll |
-----------------------------------------------------

Apa yang akan dibuat ?
- membuat controller (class) dan method default
- membuat URL menjadi lebih rapih dengan htaccess dalam mengelola URL

Kesimpulan :
- MVC adalah Model, View, Controller
- Model :
    * Merepresentasikan pengetahuan
    * Mengelola data
    * Lohika Bisnis
- Controller :
    * Perantara antara Model dan View
    * Menangani pemrosesan pada aplikasi
    * Menangani aksi dari user
- View :
    * Output, yang dilihat oleh user
    * Merepresentasikan visual dari model
    * Lapisan presentasi
